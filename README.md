箱包 AI 生图工具

ELLE箱包内部设计工作台。项目仓库：
[`mengqixi/Sino_gpt`](https://github.com/mengqixi/Sino_gpt)。

项目现状、接口、部署约定、已知风险和历史问题见
[`docs/HANDOFF.md`](docs/HANDOFF.md)。

## 主要功能

- 智能调色：自动识别五金保护区，也支持智能框选和手动画笔修正。
- AI 生成：换色、材质替换、模特展示和完全自定义生图。
- 连续修改：选择任意生成结果继续对话，并为每轮单独选择 API。
- 生成商品图：由实拍照片和可选视频生成六张标准商品图，缺少真实角度时要求补拍。
- 自动化整理：本地分类商品图和模特图，生成唯品会15张套图并导出ZIP。
- 图片处理：结果预览、缩放、手动裁剪、四宫格拆分和独立下载。
- 提示词管理：前端编辑、另存、复制、设为默认和恢复系统模板。
- 中转站配置：请求地址、模型、字段名、认证、返回路径等均可配置。
- 历史记录：保存原图、提示词、API 配置、任务状态和生成结果。
- AI 生成历史最多保留最近 50 条；新增第 51 条时自动删除最老任务及不再被引用的原图、结果图和连续修改图。失败或结果未知任务保留 7 天。

## 本地启动

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn backend.app:app --reload --host 127.0.0.1 --port 8000
```

另开终端启动前端开发服务：

```powershell
cd frontend
npm install
npm run dev
```

开发访问：`http://127.0.0.1:5173`

## Docker 启动

```powershell
docker compose build
docker compose up -d
```

访问：`http://127.0.0.1:8000`

Docker 会挂载以下目录，升级镜像时不会丢数据：

- `data/`：SQLite 数据库
- `backend/uploads/`：上传原图
- `backend/results/`：生成图和本地调色结果
- `backend/models/`：预留 SAM/SAM2 模型目录

## 2GB 内存服务器建议

“生成商品图”按低配服务器设计：视频默认由浏览器本地读取、分层抽帧并压缩到最长边 1280，只上传最多 12 张候选帧；透明底初稿、擦除和恢复也在浏览器 Canvas 完成。只有浏览器无法解码 MP4/MOV 且用户明确确认时，才逐个上传原视频交给服务器兼容处理。

生产环境建议：

- 使用 Python 3.12、单个 Uvicorn worker，不能开启 `--reload`，也不要同时运行多个应用副本处理同一 SQLite 数据库。
- 前端在开发电脑执行 `npm run build`，服务器只运行构建后的 `frontend/dist`。
- 安装 `ffmpeg` 作为 MOV/特殊编码视频的服务端兜底；HEIC/HEIF 由 `pillow-heif` 处理。
- 为进程设置 `MALLOC_ARENA_MAX=2`、`OMP_NUM_THREADS=1`、`OPENBLAS_NUM_THREADS=1`，Dockerfile 已内置这些限制。
- `data/` 必须放在持久化磁盘，不要放到内存盘。商品图上传会保留至少 3GB 空闲；源素材开始下一轮即删除，遗留素材最多保留 24 小时。
- 若 Nginx 允许 500MB 原视频兜底，请对商品图接口关闭请求缓冲或把临时目录放到磁盘；不要让 `client_body_temp_path` 或 systemd `TMPDIR` 指向 tmpfs。
- 建议配置 1–2GB swap 作为异常图片解码的最后保护，但不能用 swap 代替上述单并发和文件上限。

## 轻量服务器更新

生产服务器使用项目专属 Python 环境，前端在本地构建并将 `frontend/dist` 一同提交到 GitHub。服务器不运行 npm、Docker 或前端构建。

本地发布：

```powershell
cd frontend
npm run build
cd ..
git add -A
git commit -m "更新说明"
git push origin main
```

服务器更新：

```bash
/opt/sino-gpt/update.sh
```

更新脚本只拉取 `main` 分支并重启 `sino-gpt.service`。数据库、上传图和结果图位于 `/opt/sino-gpt/` 的独立持久化目录，不会被 Git 更新覆盖。

## 智能调色

左侧“智能调色”是本地处理，不调用中转站 API。流程：

1. 上传一张女包图。
2. 点击“自动识别”，生成包包主体和五金保护区。
3. 用“智能框选”补选五金或排除误识别，也可以使用保护/擦除画笔精修。
4. 点击调色板颜色即时预览，保护区保持原色。
5. 保存调色结果，下载或选为 AI 生成源图。

第一版使用 OpenCV 自动识别，后续可接 SAM/SAM2 增强分割。

## 中转站 API

API Base URL、模型名、接口路径、字段名、返回字段路径等都在前端 API 设置页配置。API Key 保存在后端 SQLite 中，前端读取时只显示掩码。

API 配置按用途分为两类：

- `生图 API`：只用于 AI 生图和电商生图，不能用于素材分析。
- `图文分析 API`：用于自动化整理和商品图素材分析，不能用于生图。

后端会再次校验 API 用途，即使前端参数被手动修改，也不会把图文分析 API 发送到生图接口。两类配置分别维护默认项。

名称为“快速”的启用配置会优先作为首次生成和继续修改的默认项，所有页面仍可手动切换配置。
HTTP 524 会记录为“结果未知”，系统不会自动重试，避免一次请求被重复扣费。

## 生成商品图

左侧“生成商品图”面向只有实物、没有标准商品图的场景。一次任务只处理一个“款号＋颜色”，
支持五个多图入口（正面、背面、半侧面、顶部开口、Logo）和最多三个可选 MP4/MOV 视频。

1. 上传实拍素材；每个图片入口最多 5 张、单张 30MB，单个视频最长 60 秒且不超过 500MB。
2. 选择“图文分析 API”并明确确认 1 次调用。系统从照片和本地抽取的视频帧中各选一张真实参考；缺少角度时提示补拍，不从其他视角猜测。
3. 人工确认五张参考图。分析后的补充照片按入口通过本地校验，补充视频帧由人工指定，不再次调用分析 API。
4. 选择生图 API 并确认最多 4 次调用。正面主图、背面、半侧面和顶部开口严格串行生成，每次只携带对应角度的一张参考图；失败或结果未知时暂停，不自动重试。
5. 正面透明图在本地生成初稿并支持擦除/恢复修边；Logo 细节只从真实 Logo 近照裁剪、缩放和轻度增强，不调用生图 API。
6. 人工检查包型、颜色、Logo、五金和真实配件后，可分别下载高清与 800×800 文件，或下载含 `高清/`、`800x800/` 两个目录的 ZIP。

开始下一轮会删除上一轮照片、视频、抽帧和缩略图，但保留六张结果和历史记录；异常中断的源素材最长保留 24 小时。
商品图历史按“款号＋颜色＋版本”整组展示并最多保留 50 组。旧“电商生图”页面和菜单已隐藏，旧代码暂时保留以便兼容。

## 唯品会自动化整理

左侧“自动化整理”默认使用 OpenCV 与 Pillow 在本地分析。商品原图、模特图和吊牌图分区上传后，
系统根据透明通道、主体位置、图片比例和清晰度生成15个文件槽位初稿，不会自动产生 API 费用。

素材判断按三层顺序执行：本地自动分析、可选的单次 API 分析、人工固定标签。人工标签优先级最高，
并且每个输出位置仍可单独选择来源图。API 分析只在点击“API 分析素材（1次）”时发送一次请求，
适合本地判断不准确时辅助识别正面、侧面、背面、Logo、内里、细节和透明图。

素材分析 API 在“API 设置”中新增为“图文分析 API”。自动化整理页会列出所有已启用的图文分析配置，分析前可以自由切换；生图配置不会出现在该下拉框中。

素材识别提示词会同时提供本地轻量分析结果作为参考，并要求视觉模型区分正面、半侧面/全侧面、背面和开口顶视图。人工固定标签仍拥有最高优先级。

确认后可生成符合唯品会文件名和尺寸要求的图片，并下载ZIP。`401.jpg`由商品资料表单排版，
`606.jpg`由四张来源图按固定顺序组成：正面、半侧面或全侧面、背面、开口顶视图；`801.jpg`需要上传吊牌图。

自动化整理使用独立素材表和 `data/vip_organizer/` 临时存储目录，不进入 AI 生成历史。只有刷新整个网站或点击“开始新一轮”时，才会删除上一轮素材、导出预览和 ZIP；普通页面切换、导出和下载不会删除，当前网站会话内可继续调整和重复导出。
开始新一轮时会自动删除上一轮整理素材、套图和ZIP，只保留当前一轮；AI上传图、AI结果图和历史任务不会被删除。
